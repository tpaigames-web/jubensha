# 部署到 Render（免费，固定网址，电脑不用开机）

> 为什么不用 Vercel：Vercel 是 serverless（无常驻进程），本游戏的房间数据存在服务器内存里，需要一个一直运行的进程，所以要用 Render / Railway 这类平台。Render 免费档足够。

整个过程约 10 分钟，只需要浏览器操作，代码我已经在本地 git 仓库里准备好了（含 `render.yaml` 自动配置）。

## 第一步：把代码传到 GitHub

1. 打开 https://github.com （没账号先注册一个，免费）
2. 右上角 ➕ → **New repository**，仓库名随意（如 `jubensha`），选 **Public**，点 **Create repository**
3. 上传代码，二选一：
   - **方法A（网页拖拽，最简单）**：在新仓库页面点 "uploading an existing file"，把 `C:\sohai\jubensha` 里的所有文件和文件夹拖进去（`server.py`、`render.yaml`、`requirements.txt`、`README.md`、`scripts` 文件夹、`static` 文件夹），点 Commit changes。
     ⚠️ 网页拖拽不能拖文件夹层级的话，先进仓库 → Add file → Upload files，把 scripts 和 static 里的文件分别传到对应目录（可以在文件名里输 `scripts/gallery.json` 这样自动建目录）。
   - **方法B（命令行）**：按 GitHub 新仓库页面提示，在 `C:\sohai\jubensha` 目录执行：
     ```
     git remote add origin https://github.com/你的用户名/jubensha.git
     git push -u origin master
     ```
     （首次 push 会弹出 GitHub 登录窗口）

## 第二步：在 Render 上部署

1. 打开 https://render.com → **Get Started**，直接用 GitHub 账号登录（推荐，省事）
2. 控制台点 **New +** → **Blueprint**
3. 选择你刚建的 `jubensha` 仓库 → Render 会自动读取 `render.yaml`（已配置好：新加坡节点、免费档、启动命令）→ 点 **Apply**
4. 等 1-2 分钟部署完成，你会得到一个固定网址，形如：
   **`https://jubensha-xxxx.onrender.com`**

把这个网址发给朋友，任何人、任何网络（不用同一WiFi）、任何设备浏览器打开就能玩。

> 如果不想用 Blueprint：New + → Web Service → 选仓库 → Runtime 选 Python 3 → Start Command 填 `python server.py` → 实例选 Free → 区域选 Singapore → Create。效果一样。

## 免费档注意事项

- **闲置休眠**：15 分钟没人访问会休眠，下次打开要等约 30-60 秒冷启动（页面转圈是正常的，等它醒）。开局前房主先打开网址「唤醒」它，之后整局游戏都是热的。
- **房间不保存**：房间数据在内存里，服务休眠/重新部署后房间清空——一局玩完没影响，中途休眠不会发生（一直有人在轮询就不算闲置）。
- **本地模式仍然可用**：`启动游戏.bat` 照旧，局域网玩延迟更低。两种方式并存，随便切换。

## 以后更新剧本怎么办

在 `scripts/` 里加新 JSON 后：
- 本地：重启 bat 即可
- 云端：把新文件传到 GitHub 仓库（网页 Upload files 或 `git push`），Render 会自动重新部署，约 1 分钟生效
